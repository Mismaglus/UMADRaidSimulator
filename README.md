# UMAD 走位模拟器（多机制）

一个**给我们固定队**用的 UMAD 踩塔/走位练习模拟器。纯前端、零依赖、离线可用。
顶部「机制」下拉切换不同机制，目前包含：
- **P2 · 遗弃末世 Forsaken**：脑死法（全程，含 1234 / 1256 打法）
- **P3 · 一运**：混沌之炎、混沌之水…（各机制独立练习，持续补充中）

架构：一个共享引擎（`js/core/*`：渲染 / 相机 / 移动 / 击退 / AOE 图元 / 小队列表 / 机制注册表）+ 每个副本机制一个文件（`js/mechanics/*`）。加新机制 = 新增一个文件并 `SIM.register(...)`，共享引擎只维护一份。

> ⚠️ **免责声明：不保证完全准确。**
> 站位、判定、读条时间轴、各种数值都是按我们对机制的理解**还原 / 近似**出来的，仅用于熟悉脑死法的踩塔分组和走位思路。
> **一切以游戏内实际机制和指挥为准**，发现不对的地方欢迎反馈。

## 怎么用

- **在线**：开启 GitHub Pages 后访问 <https://mismaglus.github.io/UMADRaidSimulator/>
- **离线**：下载**整个仓库**（Code → Download ZIP，解压后保持 `index.html` 和 `js/` 文件夹在一起），双击 `index.html` 即可（推荐 Chrome / Edge / Firefox）。注意：现在是多文件，单独下 `index.html` 不行，要带上 `js/` 目录。

进去后先在顶部选「**机制**」，再选好「我的职能」，点 **开始/重置** 就开始跑。

## 操作

| 操作 | 说明 |
|---|---|
| **WASD** | 移动角色（FF14 风格，无点地走位） |
| **鼠标左键拖拽** | 旋转视角 |
| **滚轮** | 缩放 |
| **重置视角 / 跟随** | 复位镜头 / 镜头跟随自己 |

## 面板选项

- **我的职能**：MT / ST / H1 / H2 / D1–D4，或 **OB（观察）= 8 人全自动**，方便从上帝视角看整套走位。
- **打法**：
  - **脑死法**：TH 踩 1·2·3·4 塔，DPS 踩 5·6·7·8 塔。
  - **1256**：TH 踩 1·2·5·6 塔，DPS 踩 3·4·7·8 塔（相当于 34 塔与 56 塔互换）。
- **移动模式**：
  - **传统**：角色面朝移动方向（按 S 转身后退）。
  - **现代**：角色面朝镜头（按 S 原地倒退）。
  - 两种模式 WASD 移动方向都跟镜头，区别只在角色朝向。
- **随机种子**（默认开）：每次点开始都换一个新点名，用来反复练不同情况；关掉则可固定种子复现同一把。
- **失败暂停 / 速度（0.25×~2×）/ 单步**：方便逐步看判定。

## 颜色与图标

- 圆柱代表玩家：**MT/ST 蓝、H1/H2 绿、D1–D4 红、BOSS 白**，头顶显示职能名。
- 头顶黄色图标 = 点名：**四角星=分摊、扇=扇形、环+点=钢铁**。
- BOSS 读条（過去/未来の終焉、消滅の脚）在 **boss 头顶**和**屏幕顶端**各显示一条，跟随视角时也能看到。

## 判定规则

按**有没有人吃到伤害（死人）**来判定：某人同时吃到 ≥2 个 AOE、分摊不足 3 人、塔内人数 ≠ 2、站在消滅の脚命中半场、或者读条结束时还有残留咏唱危机层 → 判定失败。没死人就继续。
